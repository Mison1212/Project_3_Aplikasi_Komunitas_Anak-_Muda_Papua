	ABSTRAK
	
	
	Perkembangan teknologi informasi saat ini menjadi solusi utama dalam mengatasi ketimpangan akses informasi, termasuk dalam pemetaan potensi sumber daya manusia anak muda di Provinsi Papua. Tantangan utama yang dihadapi adalah belum adanya platform digital yang terpusat untuk mendata komunitas dan menyediakan informasi pusat karir bagi talenta muda dari berbagai kabupaten di Papua. Penelitian ini bertujuan untuk merancang dan membangun "Platform Digital Komunitas & Pusat Karir Anak Muda Papua" berbasis mobile menggunakan framework Flutter. Aplikasi ini dikembangkan dengan mengintegrasikan dua teknologi database, yaitu Firebase Authentication yang berfungsi sebagai sistem keamanan autentikasi pengguna berbasis cloud, dan database MySQL yang digunakan untuk manajemen data profil pengguna secara relasional pada server lokal melalui perantara API (Application Programming Interface) berbasis PHP. Metode pengembangan sistem yang digunakan dalam penelitian ini mengikuti siklus hidup pengembangan perangkat lunak yang sistematis. Hasil dari penelitian ini adalah sebuah aplikasi Android yang mampu memvalidasi akun pengguna melalui Firebase dan melakukan sinkronisasi data profil seperti nama lengkap dan asal kabupaten ke dalam database MySQL secara real-time. Dengan adanya platform ini, proses pendataan anak muda Papua menjadi lebih terstruktur dan memudahkan pengelompokan komunitas berdasarkan wilayah, sehingga diharapkan dapat mempercepat penyebaran informasi karir dan pemberdayaan anak muda di tanah Papua.
	Kata Kunci: Flutter, Firebase Authentication, MySQL, Pusat Karir, Papua, Mobile Application. 
	
	
	
	
	
	
	
	
	ABSTRACT
	
	
	Current advancements in information technology have become the primary solution for addressing disparities in access to information, including in mapping the human resource potential of young people in Papua Province. The main challenge faced is the lack of a centralized digital platform to register communities and provide career center information for young talent from various regencies in Papua. This study aims to design and build a mobile-based “Papua Youth Community & Career Center Digital Platform” using the Flutter framework. This application was developed by integrating two database technologies: Firebase Authentication, which functions as a cloud-based user authentication security system, and a MySQL database used for relational management of user profile data on a local server via a PHP-based API (Application Programming Interface). The system development method used in this research follows a systematic software development life cycle. The result of this research is an Android application capable of validating user accounts via Firebase and synchronizing profile data—such as full names and home districts—into the MySQL database in real-time. With this platform, the process of collecting data on Papua’s youth becomes more structured and facilitates community grouping by region, thereby accelerating the dissemination of career information and the empowerment of young people in Papua.
	
	Keywords: Flutter, Firebase Authentication, MySQL, Career Center, Papua, Mobile Application.

	Pengertian Sistem 
	Sistem merupakan pondasi utama dalam pembangunan suatu aplikasi atau platform digital. Untuk memahami konsep ini secara mendalam, berikut adalah beberapa pengertian sistem menurut para ahli:
	Menurut Jeperson Hutahaean (2015:2), sistem adalah suatu jaringan kerja dari prosedur-prosedur yang saling berhubungan, berkumpul bersama-sama untuk melakukan suatu kegiatan atau menyelesaikan suatu sasaran tertentu.
	Menurut Tata Sutabri (2016:10), sistem secara sederhana dapat didefinisikan sebagai sekelompok elemen yang terintegrasi dengan maksud yang sama untuk mencapai suatu tujuan.
	Menurut Sri Mulyani (2016:2), sistem adalah sekumpulan subsistem, komponen, atau elemen yang saling bekerja sama dengan tujuan yang sama untuk menghasilkan informasi yang berguna atau mencapai target tertentu.
	Berdasarkan ketiga definisi di atas, maka dapat disimpulkan bahwa sistem adalah kesatuan dari berbagai komponen, elemen, atau prosedur yang saling terintegrasi, berinteraksi, dan bekerja sama secara terarah untuk memproses suatu masukan (input) menjadi keluaran (output) demi mencapai tujuan tertentu yang telah ditetapkan.
	Aplikasi Mobile (Mobile Application)
	Aplikasi mobile adalah perangkat lunak yang dirancang secara khusus untuk dijalankan pada perangkat genggam seperti ponsel pintar atau tablet.
	Framework Flutter dan Bahasa Pemrograman Dart
	Flutter merupakan perangkat pengembangan pencitraan antarmuka (UI SDK) open-source yang dikembangkan oleh Google untuk membangun aplikasi multiplatform dari satu basis kode tunggal. Flutter menggunakan bahasa pemrograman Dart.
	Firebase
	Firebase adalah platform penyedia layanan backend (Backend-as-a-Service atau BaaS) dari Google yang mempermudah pengembang dalam mengelola database (Firestore), autentikasi (Firebase Auth), dan notifikasi (Firebase Cloud Messaging).
	Unified Modeling Language (UML)
	UML adalah bahasa standar untuk menspesifikasikan, memvisualisasikan, membangun, dan mendokumentasikan artefak dari sistem perangkat lunak berorientasi objek (OOAD).

	Membantu Ketua dalam menyusun laporan pertanggungjawaban kegiatan berkala.
	Berkoordinasi dengan divisi terkait dalam menyusun jadwal kegiatan pendataan atau sosialisasi platform digital kepada anggota.
	Bendahara
	Mengatur dan mencatat seluruh arus keuangan masuk maupun keluar yang digunakan untuk mendanai operasional dan kegiatan komunitas.
	Menyusun laporan keuangan organisasi secara berkala secara transparan dan akuntabel.
	Bertanggung jawab atas pengelolaan anggaran pengadaan fasilitas teknis pendukung pengembangan platform komunitas.
	Koordinator Divisi Hubungan Komunitas
	Bertindak sebagai administrator utama yang menjembatani hubungan komunikasi dan penyebaran informasi secara vertikal kepada koordinator wilayah di tiap kabupaten.
	Mengelola proses verifikasi awal terhadap keabsahan data profil anggota komunitas berskala luas yang masuk ke dalam sistem.
	Bertanggung jawab memantau keaktifan forum komunitas digital serta memelihara stabilitas interaksi antar-anggota didalam platform.
	Koordinator Wilayah Kabupaten
	Bertanggung jawab sebagai perwakilan resmi pengurus pusat komunitas di tingkat daerah administratif kabupaten masing-masing.
	Menggerakkan, mendata, dan membantu pemuda-pemudi di wilayah kabupatennya agar dapat mendaftarkan diri secara valid ke dalam sistem.
	Menyaring dan menyalurkan kembali informasi lowongan pekerjaan dari pusat karir agar dapat diakses secara merata oleh anggota yang berada di daerah kabupatennya.
	
	 Tata Laksana Sistem yang Berjalan
	Sistem pendataan anggota dan penyebaran informasi lowongan kerja yang berjalan saat ini masih menggunakan cara konvensional. Anggota harus mengisi formulir manual atau Google Form secara terpisah di masing-masing grup daerah, lalu pengurus merekapnya ke dalam spreadsheet lokal. Informasi lowongan kerja dibagikan secara acak melalui grup chat media sosial (seperti WhatsApp atau Telegram).
	
	Gambar 3.2
	 Masalah yang Dihadapi
	Berdasarkan analisis tata laksana sistem yang berjalan, ditemukan beberapa kelemahan utama, yaitu:
	Penyimpanan Data Terfragmentasi: Data profil anggota terpisah-pisah di berbagai admin kabupaten, sehingga tidak ada basis data relasional pusat untuk memetakan total potensi SDM pemuda Papua.
	Kerawanan Keamanan Akun: Tidak adanya sistem autentikasi terpusat membuat data rentan dimanipulasi dan disalahgunakan oleh pihak luar.
	Penumpukan Informasi (Information Overload): Penyebaran lowongan kerja di grup media sosial menumpuk dan tercampur dengan pesan obrolan biasa, sehingga menyulitkan pencarian data tanpa adanya sistem filter halaman (pagination) yang teratur.
	
	 Alternatif Pemecahan Masalah
	Untuk mengatasi kendala pada sistem yang berjalan, diajukan 2 (dua) alternatif pemecahan masalah sebagai berikut: 
	Alternatif I: Membangun sistem informasi berbasis Web responsive untuk pendataan terpusat menggunakan server internal. Namun, kelemahannya adalah aksesibilitas pemuda di daerah kurang fleksibel karena keterbatasan perangkat PC/laptop di lapangan.
	Alternatif II (Solusi yang Dipilih): Membangun Platform Digital Berbasis Mobile Application (Android) menggunakan framework Flutter. Mengintegrasikan Firebase Authentication untuk keamanan login pengguna di cloud, serta menyinkronkan data profil (Nama & Kabupaten) ke database relasional MySQL via API PHP. Sistem dilengkapi fitur penayangan bursa kerja yang mengimplementasikan teknik pagination agar aplikasi tetap ringan dijalankan.

	BAB IV
	RANCANGAN SISTEM YANG DIUSULKAN
	
	 Usulan Prosedur yang Baru
	Usulan prosedur yang baru pada aplikasi Papua Youth Career App atau Karir Muda Papua dirancang untuk memperbaiki proses penyampaian informasi lowongan pekerjaan, pendaftaran lowongan, pengelolaan data user, dan pelaporan lamaran secara digital. Sistem yang diusulkan menggantikan proses manual menjadi proses berbasis aplikasi mobile yang terhubung dengan Firebase Authentication, API PHP, dan database MySQL.
	Pada prosedur baru ini terdapat dua aktor utama, yaitu User dan Admin. User merupakan pengguna umum yang dapat melakukan registrasi akun, verifikasi email, login, mengelola data pribadi, mencari lowongan, mendaftar lowongan, dan melihat status lamaran. Admin merupakan pengguna dengan hak akses khusus yang dapat mengelola data user, data kabupaten, data spesifikasi karir, data lowongan, melihat laporan pendaftaran, dan mengubah status lamaran user.
	Dengan prosedur baru ini, setiap user yang melakukan registrasi tidak langsung dapat masuk ke sistem sebelum melakukan verifikasi email. Setelah email diverifikasi, user dapat login dan menggunakan fitur aplikasi. Ketika user mendaftar lowongan, data lamaran akan tersimpan ke database dan dapat dilihat oleh admin. Admin kemudian dapat memproses lamaran tersebut dengan mengubah status menjadi pending, accepted, atau rejected. Status tersebut dapat dilihat kembali oleh user melalui halaman lamaran.
	Pemisahan hak akses dilakukan berdasarkan field role pada tabel users. Setiap user baru yang melakukan registrasi akan otomatis memiliki role user. Admin ditentukan melalui perubahan role pada database menjadi admin. Keamanan API diperkuat dengan penggunaan Firebase ID Token, sehingga endpoint yang bersifat sensitif hanya dapat diakses oleh pengguna yang sudah login dan memiliki hak akses sesuai:
	
	Prosedur Registrasi User yang Diusulkan
	Prosedur registrasi user yang diusulkan adalah sebagai berikut:
	User membuka halaman registrasi pada aplikasi.
	User mengisi nama lengkap, asal kabupaten, spesifikasi karir, email, dan kata sandi.
	Sistem memvalidasi kelengkapan data registrasi.
	Data akun dikirim ke Firebase Authentication.
	Firebase membuat akun baru dan mengirim email verifikasi kepada user.
	Data profil user dikirim melalui API PHP dan disimpan ke database MySQL.
	User diarahkan ke halaman login.
	User membuka email dan melakukan verifikasi akun.
	User dapat login setelah email berhasil diverifikasi.
	
	Prosedur Login yang Diusulkan
	Prosedur login yang diusulkan adalah sebagai berikut :
	User atau Admin membuka halaman login.
	User atau Admin memasukkan email dan kata sandi.
	Sistem memvalidasi akun melalui Firebase Authentication.
	Sistem memeriksa apakah email sudah diverifikasi.
	Jika email belum diverifikasi, sistem menolak login dan meminta user melakukan verifikasi email.
	Jika email sudah diverifikasi, sistem memeriksa role pengguna.
	Jika role bernilai user, pengguna masuk ke halaman utama user.
	Jika role bernilai admin, pengguna dapat mengakses menu admin.
	
	Prosedur Kelola Profil User yang Diusulkan
	Prosedur kelola profil user yang diusulkan adalah sebagai berikut:
	User login ke aplikasi.
	User membuka halaman profil.
	User mengubah data pribadi seperti nama lengkap, asal kabupaten, nomor telepon, dan spesifikasi karir.
	Sistem mengirim Firebase ID Token dan data profil ke API PHP.
	API PHP memverifikasi token user.
	API PHP menyimpan perubahan data ke tabel users.
	Sistem menampilkan pesan bahwa profil berhasil disimpan.
	
	Prosedur Pendaftaran Lowongan yang Diusulkan
	Prosedur pendaftaran lowongan yang diusulkan adalah sebagai berikut:
	User login ke aplikasi.
	User membuka halaman lowongan.
	User mencari dan memilih lowongan pekerjaan.
	User membuka detail lowongan.
	User menekan tombol daftar lowongan.
	Sistem mengirim data pendaftaran ke API PHP.
	API PHP menyimpan data lamaran ke tabel applications dengan status awal pending.
	Data lamaran dapat dilihat oleh admin pada menu laporan lamaran.
	
	Prosedur Pengelolaan Lamaran oleh Admin yang Diusulkan
	Prosedur pengelolaan lamaran oleh admin yang diusulkan adalah sebagai berikut:
	Admin login ke aplikasi.
	Admin membuka halaman admin.
	Admin melihat laporan pendaftaran lowongan.
	Admin melihat data pelamar, lowongan yang dilamar, status lamaran, dan nomor telepon pelamar.
	Admin mengubah status lamaran menjadi pending, accepted, atau rejected.
	Sistem menyimpan perubahan status ke tabel applications.
	User dapat melihat hasil perubahan status pada halaman lamaran.
	
	Prosedur Pengelolaan Data Master oleh Admin yang Diusulkan
	Prosedur pengelolaan data master oleh admin yang diusulkan adalah sebagai berikut:
	Admin login ke aplikasi.
	Admin membuka halaman admin.
	Admin dapat menambahkan data kabupaten.
	Admin dapat menambahkan data spesifikasi karir.
	Admin dapat menambahkan, mengubah, dan menghapus data lowongan.
	Data yang dikelola admin tersimpan ke database MySQL melalui API PHP.

	Rancangan Prototype/Tampilan
	Aplikasi Karir Muda Papua yang dibuat sebagai gambaran awal antarmuka sistem sebelum digunakan oleh user dan admin. Rancangan prototype dibuat untuk memudahkan pengguna dalam memahami alur penggunaan aplikasi, mulai dari proses registrasi, login, pencarian lowongan, pendaftaran lowongan, sampai pengelolaan data oleh admin.
	Tampilan aplikasi dirancang dengan konsep sederhana, modern, dan mudah digunakan. Warna utama yang digunakan adalah hijau toska atau teal yang memberikan kesan profesional, segar, dan sesuai dengan tema aplikasi karir. Setiap halaman dibuat dengan tata letak yang rapi, penggunaan tombol yang jelas, serta form input yang mudah dipahami.
	Rancangan tampilan pada aplikasi ini dibagi menjadi dua bagian utama, yaitu tampilan untuk user dan tampilan untuk admin.
	
	Tampilan user terdiri dari beberapa halaman utama, yaitu:
	Tampilan Login
	Halaman login digunakan oleh user untuk masuk ke dalam aplikasi menggunakan email dan kata sandi. Pada halaman ini terdapat form email, form kata sandi, tombol masuk, serta pilihan untuk menuju halaman registrasi.
	Tampilan Registrasi
	Halaman registrasi digunakan oleh user baru untuk membuat akun. Data yang diisi meliputi nama lengkap, email, asal kabupaten, spesifikasi karir atau keahlian, dan kata sandi. Setelah registrasi, user harus melakukan verifikasi email terlebih dahulu sebelum dapat login ke aplikasi.
	Tampilan Beranda User
	Halaman beranda menampilkan informasi ringkas mengenai user, seperti nama, kabupaten, spesifikasi karir, jumlah lamaran yang dikirim, jumlah lamaran yang diterima, akses cepat ke lowongan, serta daftar lowongan terbaru.
	Tampilan Lowongan
	Halaman lowongan digunakan user untuk melihat daftar lowongan pekerjaan yang tersedia. User dapat melihat informasi lowongan seperti judul pekerjaan, nama perusahaan, lokasi, kategori, deadline, dan status apakah lowongan sudah didaftari atau belum.
	Tampilan Detail Lowongan
	Halaman detail lowongan menampilkan informasi lengkap mengenai lowongan pekerjaan, seperti nama pekerjaan, perusahaan, lokasi, kategori, deadline, gaji, deskripsi pekerjaan, persyaratan, dan tombol untuk mendaftar lowongan.
	Tampilan Lamaran User
	Halaman ini digunakan user untuk melihat daftar lamaran yang telah dikirim. Pada halaman ini user dapat mengetahui status lamaran, yaitu menunggu, diterima, atau ditolak.
	Tampilan Profile User
	Halaman profil digunakan user untuk melihat dan mengubah data pribadi. Data yang dapat diubah meliputi nama lengkap, nomor telepon, asal kabupaten, dan spesifikasi karir. User juga dapat keluar dari akun melalui halaman ini.
	Tampilan admin terdiri dari beberapa halaman utama, yaitu:
	Tampilan Dashboard Admin
	Halaman dashboard admin menampilkan ringkasan data sistem, seperti total user, total lowongan, total lamaran, jumlah lamaran diterima, ringkasan status lamaran, serta daftar lamaran terbaru.
	Tampilan Kelola Lowongan
	Halaman ini digunakan admin untuk mengelola data lowongan. Admin dapat menambah, mengedit, dan menghapus lowongan pekerjaan. Data lowongan meliputi judul lowongan, nama perusahaan, gaji, lokasi, kategori, deadline, deskripsi, dan persyaratan.
	Tampilan Kelola Lamaran
	Halaman ini digunakan admin untuk melihat seluruh lamaran yang dikirim oleh user. Admin dapat mengubah status lamaran menjadi diterima, ditolak, atau dikembalikan ke status menunggu.
	Tampilan Data User
	Halaman data user digunakan admin untuk melihat daftar user yang telah terdaftar di aplikasi. Informasi yang ditampilkan meliputi nama user, kabupaten, spesifikasi karir, email, dan nomor telepon. Nomor telepon ditampilkan agar admin dapat menghubungi user apabila lamaran diterima.
	Tampilan Data Master
	Halaman data master digunakan admin untuk mengelola data kabupaten dan spesifikasi karir. Data ini digunakan sebagai pilihan pada saat user melakukan registrasi atau mengedit profil.
	Dengan adanya rancangan prototype ini, proses pengembangan aplikasi menjadi lebih terarah karena setiap tampilan sudah memiliki fungsi dan alur penggunaan yang jelas. Prototype juga membantu memastikan bahwa kebutuhan user dan admin dapat diterapkan ke dalam sistem aplikasi Karir Muda Papua.
	
	Implementasi Sistem yang Diusulkan
	Implementasi sistem yang diusulkan merupakan tahapan penerapan rancangan sistem ke dalam bentuk aplikasi yang dapat digunakan. Aplikasi Karir Muda Papua diimplementasikan menggunakan Flutter sebagai framework pengembangan aplikasi, Firebase Authentication sebagai layanan autentikasi user, PHP sebagai API backend, dan MySQL sebagai database penyimpanan data.
	Sistem yang diusulkan memiliki dua jenis pengguna, yaitu user dan admin. User dapat melakukan registrasi, verifikasi email, login, mengelola profil, melihat lowongan, mendaftar lowongan, dan melihat status lamaran. Admin dapat mengelola data user, data kabupaten, data spesifikasi karir, data lowongan, serta mengubah status lamaran yang dikirim oleh user.
